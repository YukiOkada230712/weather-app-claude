# PR レビューコマンド

以下の観点でコードレビューを行ってください：

1. **型安全性**
   - any 型が使われていないか
   - 関数の引数・戻り値に型定義があるか

2. **Atomic Design 準拠**
   - コンポーネントが適切な階層（atoms/molecules/organisms）に配置されているか

3. **テスト**
   - 新規コンポーネントにテストが追加されているか
   - テストカバレッジは十分か

4. **エラーハンドリング**
   - API エラーがユーザーに通知されているか

レビュー結果は以下の形式で出力してください：
- ✅ 問題なし
- ⚠️ 改善推奨（理由と修正案）
- ❌ 要修正（理由と修正案）
